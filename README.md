# Ведомость AI — прототип (итерация 1)

AI-агент ведения преподавательских ведомостей в Telegram. Часть будущего «большого
бота преподавателя», где Ведомость AI — один из инструментов (субагент).

## Что уже есть (каркас)
- **Доменное ядро** (`core/`) — вызывается ботом и, в будущем, ботом-оркестратором:
  - `models.py` — модель данных с инвариантами плана: **append-only оценок**,
    статусный автомат, lineage ведомостей, маркер «академ», версии GradingScheme;
  - `statuses.py` — конечный автомат статусов (Черновик → Заполняется → Закрыта → …);
  - `services/statement_service.py` — создание ведомости, append-only запись оценок,
    актуальный срез (последняя оценка по каждому элементу);
  - `services/grading_service.py` — **заглушка** расчётного движка (ждёт данные ПУД);
  - `parsing/` — **заглушки** границ с ИИ: текст (AI Studio), голос (SpeechKit), ПУД.
- **Бот** (`bot/main.py`, aiogram 3) — `/start`, меню, тестовая группа, приём текста/голоса
  (пока с пояснениями-заглушками там, где нужны данные/права).
- **Тестовая группа** (`seed/test_group.py`) — 20 выдуманных студентов (без ПДн).

## Запуск (локально)
```bash
cd app
python -m venv .venv
.venv\Scripts\activate           # Windows
pip install -r requirements.txt
copy .env.example .env           # заполнить TELEGRAM_BOT_TOKEN
python -m bot.main
```

## Стек и инфраструктура
- Прототип: SQLite + локальный запуск. Боевой контур: Managed PostgreSQL + Yandex Cloud
  (меняется только `DATABASE_URL`). Yandex Cloud уже доступен (SA `leonov-deployer`,
  folder `b1gvtru3guuc1oipcs4p`, роль `editor`, биллинг активен).
- Голос — Yandex SpeechKit; разбор текста/ПУД — Yandex AI Studio (включаются по мере
  выдачи ролей `ai.*` и получения примеров данных).

## Чего пока нет намеренно (ждём данные — см. договорённость с заказчиком)
Расчётный движок, реальные форматы ПУД и Excel-выгрузки, экспорт в СЭВ/ЭП, дашборд
академрука, роли ассистента/старосты, пересдачи — наращиваются поверх ядра без переделки.
